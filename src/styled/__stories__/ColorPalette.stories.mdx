## Colors
There are two primary goals in how our UI uses color:
1. Create a pleasing visual aesthetic.
1. Communicate meaning clearly and consistency.

DDS therefore has two sets of color tokens, one for each of these goals.

### Color Tokens
Color tokens define the space of all the colors that are available within our UI. They do this by mapping to specific
hex values.

This mapping reduces the number of possible colors from 16 million down to 60. If this smaller list of colors is
well-chosen, none of the available colors should clash with each other, and we should have all the colors we need to
build UIs.

### Semantic Tokens
Semantic tokens ensure a consistent use of color for a given meaning within the UI. They do this by mapping to color tokens.

As an example, <code>content</code> and <code>background</code> are the default semantic tokens for text and background
colors, respectively. With all component styles using these tokens to color their text and backgrounds, it is possible
to create an alternate color scheme like dark mode, where all text is light and all backgrounds are dark. Because the
semantic tokens are defined in one place, we can make one change and have it impact the UI in a predictable manner.

### Usage
For this system of color to work, we must follow a two rules:

1. <b>Color token definitions are the only place in our UI where hex values should exist. All other usage of color must
reference the color tokens rather than hex values directly.</b>
1. <b>UI elements and their styles should ONLY reference semantic tokens.</b>
